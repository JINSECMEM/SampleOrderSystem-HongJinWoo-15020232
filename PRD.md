## 1. 목적

반도체 시료의 주문 접수부터 출고까지 전 과정을 콘솔 앱으로 관리한다.  
담당자가 주문 상태, 재고, 생산 현황을 한 화면에서 파악하고 처리할 수 있어야 한다.

---

## 2. 주문 상태 흐름

```
[주문 접수] ──► RESERVED
                   │
          ┌────────┴────────┐
          ▼                 ▼
       REJECTED          재고 확인
                      ┌────┴────┐
                   충분        부족
                      │         │
                  CONFIRMED  PRODUCING
                      │         │
                      │    생산 완료
                      │         │
                      └────┬────┘
                           ▼
                        RELEASE
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수, 승인 대기 |
| `REJECTED` | 주문 거절 (유효하지 않은 주문) |
| `CONFIRMED` | 승인 완료, 출고 대기 중 |
| `PRODUCING` | 승인 완료, 재고 부족으로 생산 중 |
| `RELEASE` | 출고 완료 |

## 3. 기능 명세

### 3.1 시료 관리

| 기능 | 설명 |
|------|------|
| 시료 등록 | 이름, 평균 생산 시간(분), 수율을 입력하여 신규 시료를 시스템에 추가 |
| 시료 조회 | 시료 ID로 단건 조회 — ID, 이름, 평균 생산 시간, 수율, 현재 재고 표시 |
| 시료 목록 검색 | 전체 목록 출력; 이름 등 속성으로 필터링; 현재 재고 수량 함께 표시 |

> **수율(yield_rate)** = 정상 생산 시료 수 / 총 생산 시료 수 (0 < yield_rate ≤ 1.0)

---

### 3.2 주문 처리

1. 주문 접수 → 상태 `RESERVED`
2. 담당자가 콘솔 입력으로 **승인 / 거절** 선택
   - 거절 → `REJECTED` (이후 처리 없음)
   - 승인 → 재고 확인
     - 재고 충분 → `CONFIRMED` → 즉시 출고 처리 가능
     - 재고 부족 → `PRODUCING` (생산 큐에 등록)

---

### 3.3 모니터링

담당자 현황판. 다음 정보를 한 화면에 표시한다.

**주문량 확인**
- 현재 상태별 주문 목록: `RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`
- `REJECTED`는 표시하지 않는다.

**재고 현황**

| 재고 상태 | 조건 |
|-----------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 현재 재고 = 0 |

각 시료별 현재 재고 수량과 위 상태를 함께 표시한다.

---

### 3.4 생산 라인

**생산량 계산**

```
실생산량 = ceil(부족분 / (수율 × 0.9))
총생산시간(분) = 평균생산시간 × 실생산량
```

**스케줄링**: FIFO (선입선출) 생산 큐

**표시 정보**
- 현재 생산 중인 시료: 주문 정보, 현재까지의 생산량
- 대기 중인 생산 큐 목록

**생산 완료 시**: `PRODUCING` → `CONFIRMED` 자동 전환

---

### 3.5 출고 처리

- `CONFIRMED` 상태의 특정 주문을 선택하여 출고 실행
- 출고 완료 시 상태 → `RELEASE`
- 재고에서 주문 수량만큼 차감
